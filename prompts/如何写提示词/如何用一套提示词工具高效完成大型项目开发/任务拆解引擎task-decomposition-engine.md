# 定位

**定位：把一个大目标炸开成可执行的任务树。**

超大型项目最常见的失败模式不是技术问题，而是"不知道下一步该做什么"。AI 最擅长做的事情之一就是把一个宏观目标分解成结构化的任务清单，但前提是你给它正确的约束。

**为什么不可或缺：** 没有任务拆解，大型项目会变成"想到哪写到哪"。AI 会跟着你的即兴提问走，而不是沿着一条有规划的路径推进。任务拆解引擎确保每一轮对话都在推进真实进度。

# 提示词

我有一个项目目标。请你帮我把它拆解成一份可执行的结构化任务清单。

你必须严格按照以下层级拆解：

第一层：项目阶段（Phase）
- 把项目分成 3-6 个有明确交付物的阶段
- 每个阶段用一句话定义"这个阶段做完意味着什么"

第二层：里程碑（Milestone）
- 每个阶段拆出 2-4 个里程碑
- 每个里程碑必须有可验证的完成标准（不是"基本完成"，而是"什么东西能跑通"）

第三层：具体任务（Task）
- 每个里程碑拆出具体的实现任务
- 每个任务必须满足：一个人在一次 AI 会话中可以完成
- 标注任务之间的依赖关系（哪些必须先做，哪些可以并行）

第四层：风险标注
- 对每个阶段标注最可能的卡点
- 标注哪些任务如果延期会影响整体进度（关键路径）

输出格式：
阶段 → 里程碑 → 任务清单（含依赖和优先级）→ 风险标注

规则：
- 任务粒度必须足够小，每个任务的描述能直接作为一次 AI 对话的输入
- 不要输出"需要进一步讨论"之类的占位任务
- 如果某个环节你判断存在技术选型分歧，直接标注为"决策点"并给出候选方案
- 所有任务必须有明确的先后顺序或并行标注
